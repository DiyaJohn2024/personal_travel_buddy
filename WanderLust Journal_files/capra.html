Build a clean, simple frontend for an app called “SaveTheSpot – Your Personal Travel Memory”.

Important:

  • Use Firebase Authentication ONLY (Email/Password).
  • Do NOT use Firestore or any database.
  • Use mock JSON data stored inside the app to represent Trips and Memories.
  • All add/edit/delete operations should update the local JSON or local state only.

Features:

  1. Sign Up and Login screens (email + password using Firebase Auth).
  2. Dashboard showing a list of trips from mock data.
  3. "Add Trip" form (location, date, category) → add to local list.
  4. Trip Details page showing memories from mock data filtered by tripId.
  5. "Add Memory" form (title, description, photo upload mock, tags).
  6. Memory cards displayed in a grid.
  7. Simple navigation (Home, Trips, Logout).

UI Suggestions:

  • Minimal, travel-themed design.
  • Trip cards: location, date, and category badge.
  • Memory cards: image + title + description.
  • Tags as small chips.
  • Clean forms with rounded inputs and soft shadows.

Behavior:

  • After login → load the mock JSON data.
  • Store new trips/memories in local state (not in database).
  • Logout should clear state and return to login.

Goal: A fully working demo UI using Firebase only for login and mock JSON for all data.
